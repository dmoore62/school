README File

Included Files:

LA.c - Source code for lexical analyzer
LA.out - compiled object code from "gcc -o LA.out LA.c"
test.in - File that LA.c reads in and tokenizes
output.txt - File LA.c writes to during runtime

To Run File:

Compile with "gcc -o [outFile] LA.c"
Make sure input file is in the same folder and named "file.in"
Type "./LA.out"

During runtime:

All error are printed to console, ant errors means that tokens are corrupt
The read source code, list of tokens, and lexeme are printed to "output.txt"
